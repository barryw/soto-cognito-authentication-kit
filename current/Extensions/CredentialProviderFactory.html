    Credential provider using Cognito userpool authentication

    You can authenticate with Cognito UserPools with various methods. Options available include .password and .refreshToken. If you import SotoCognitoAuthenticationSRP you also get secure remote password authentication with .srp.

    When authenticating, Cognito might respond with a challenge. Many of these challenges require user input. The respondToChallenge closure allows you to provide challenge response parameters. The respond to challenge closure is called with a challenge type the related input parameters, an error is the last challenge response produced an error and the EventLoop everything is running on. If you return nil that is considered a failed challenge and an error will be thrown. Below is a list of common challenges with the expected parameters to be returned. .newPasswordRequired: requires NEW_PASSWORD .smsMfa: SMS_MFA_CODE
